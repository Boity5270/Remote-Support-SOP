TechFlow Remote Support — Standard Operating Procedure (SOP)

1. Purpose

The purpose of this SOP is to ensure TechFlow employees receive consistent, efficient, and secure remote troubleshooting and end-user support.

2. Scope

This SOP applies to all TechFlow IT support staff providing Tier 1 and Tier 2 assistance to remote and hybrid employees. It covers troubleshooting, escalation, documentation, and communication procedures for remote IT support.

3. Tools Required

  • Remote Desktop Access: AnyDesk, TeamViewer
  • Communication/Video: Microsoft Teams, Zoom
  • Ticketing: Freshdesk (logging, tracking, closing)
  • Security & Authentication: Multi-Factor Authentication (MFA)

4. Standard Troubleshooting Process

  1. Confirm User Identity
    • Authenticate users with HR records. Only authorized employees receive assistance.
  2. Gather Details
    • Collect device specs (laptop/desktop, OS, version) and request screenshots or error codes.
    • Understand urgency and business impact.
  3. Apply Troubleshooting Script
    • Use scenario-based scripts (VPN, password reset, Outlook sync) and follow a structured approach.
  4. Document Actions in Ticket Log
    • Record all steps taken, tools used, and outcomes in Freshdesk.
  5. Confirm Resolution with User
    • Test functionality live with the user and ensure issue is resolved before closure.
  6. Close Ticket & Update Knowledge Base
    • Mark ticket resolved and add new solutions to the internal knowledge base.

5. Escalation Triggers

  • Unresolved Issues: Problem remains unresolved after Tier 1 script execution.
  • Hardware Failures: Device requires onsite replacement or repair.
  • Security Incidents: Suspicious logins, potential breaches, phishing alerts.
  • Widespread Impact: More than 5 users affected by the same recurring issue.

Troubleshooting Scripts

VPN Troubleshooting

  1. Confirm user identity and check VPN client version.
  2. Verify Internet connectivity and restart VPN client.
  3. Check for local firewall or AV blocking the connection.
  4. If required, collect VPN logs and escalate to Tier 2.

Password Reset Script

  1. Authenticate user via HR record.
  2. Reset password in AD/IDP and force password change at next login if required.
  3. Confirm user can log in to VPN and email.

Outlook / Email Sync Script

  1. Verify network connectivity and account status.
  2. Clear cached credentials and re-add the account if necessary.
  3. Collect email client logs and escalate to Tier 2 if persists.

Support Logs

Logging Requirements

  • Ticket ID and timestamp
  • Technician name and contact
  • Actions taken (step-by-step)
  • Attachments: logs, screenshots, diagnostic outputs
  • Escalation notes (if applicable)

6. Key Performance Indicators

  • First Contact Resolution (%)
  • Mean Time to Resolve
  • Escalation Rate (%)
  • Customer Satisfaction

7. Best Practices

  • Maintain professionalism and empathy in remote interactions.
  • Obtain consent before remotely accessing user devices.
  • Apply proven troubleshooting scripts first to minimize downtime.
  • Continuously update the knowledge base from recurring tickets.
